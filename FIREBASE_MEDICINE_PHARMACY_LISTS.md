# Firebase Medicine and Pharmacy Store Lists

Implementation guide for loading medicine/tablet and pharmacy store lists in the PharmaFlow React Native user application.

## 1. Data Sources

The app must load production data from Cloud Firestore. Do not place medicine or pharmacy arrays inside screens.

```text
medicines/{medicineId}
pharmacies/{pharmacyId}
pharmacies/{pharmacyId}/inventory/{medicineId}
```

- `medicines` is the platform’s canonical medicine catalogue.
- `pharmacies` contains verified pharmacy stores.
- `inventory` is optional and must not be treated as guaranteed stock.
- Actual availability is confirmed by a pharmacy after an order request.

## 2. Medicine Document

## 5. Medicine List Service

Use cursor pagination. Do not load the complete medicine catalogue.

```ts
import firestore, {
  FirebaseFirestoreTypes,
} from '@react-native-firebase/firestore';

const PAGE_SIZE = 20;

export interface MedicinePage {
  items: Medicine[];
  cursor: FirebaseFirestoreTypes.DocumentSnapshot | null;
  hasMore: boolean;
}

export async function getMedicines(
  cursor?: FirebaseFirestoreTypes.DocumentSnapshot | null,
): Promise<MedicinePage> {
  let query = firestore()
    .collection('medicines')
    .where('status', '==', 'ACTIVE')
    .orderBy('normalizedName')
    .limit(PAGE_SIZE);

  if (cursor) {
    query = query.startAfter(cursor);
  }

  const snapshot = await query.get();

  return {
    items: snapshot.docs.map(document => ({
      id: document.id,
      ...document.data(),
    } as Medicine)),
    cursor: snapshot.docs.at(-1) ?? null,
    hasMore: snapshot.size === PAGE_SIZE,
  };
}
```

## 6. Medicine Search

Firestore does not provide full-text search. Prefix queries can support basic medicine and generic-name searches.

```ts
const normalizeSearch = (value: string) =>
  value.trim().toLocaleLowerCase();

export async function searchMedicines(term: string): Promise<Medicine[]> {
  const normalized = normalizeSearch(term);

  if (normalized.length < 2) {
    return [];
  }

  const snapshot = await firestore()
    .collection('medicines')
    .where('status', '==', 'ACTIVE')
    .orderBy('normalizedName')
    .startAt(normalized)
    .endAt(`${normalized}\uf8ff`)
    .limit(20)
    .get();

  return snapshot.docs.map(document => ({
    id: document.id,
    ...document.data(),
  } as Medicine));
}
```

For searching medicine name, generic name, and brand simultaneously, use Algolia, Typesense, or a Firebase Search extension. Do not download every medicine and filter it on the device.

## 9. Pharmacy Store List

The normal store list can be loaded by city:

```ts
const PHARMACY_PAGE_SIZE = 15;

export async function getPharmacyStores(
  city: string,
  cursor?: FirebaseFirestoreTypes.DocumentSnapshot | null,
): Promise<PharmacyPage> {
  let query = firestore()
    .collection('pharmacies')
    .where('verificationStatus', '==', 'APPROVED')
    .where('status', '==', 'ACTIVE')
    .where('serviceCities', 'array-contains', city)
    .orderBy('rating', 'desc')
    .limit(PHARMACY_PAGE_SIZE);

  if (cursor) {
    query = query.startAfter(cursor);
  }

  const snapshot = await query.get();

  return {
    items: snapshot.docs.map(document => ({
      id: document.id,
      ...document.data(),
    } as PharmacyStore)),
    cursor: snapshot.docs.at(-1) ?? null,
    hasMore: snapshot.size === PHARMACY_PAGE_SIZE,
  };
}
```

Only pharmacies with both of these values may appear in the customer app:

```text
verificationStatus = APPROVED
status = ACTIVE
```

## 10. Nearby Pharmacy Query

Install:

```bash
npm install geofire-common
```

Query geohash ranges and verify actual distance on the client:

```ts
import {
  distanceBetween,
  geohashQueryBounds,
} from 'geofire-common';

export async function getNearbyPharmacies(
  latitude: number,
  longitude: number,
  radiusKm: number,
): Promise<PharmacyStore[]> {
  const center: [number, number] = [latitude, longitude];
  const radiusMeters = radiusKm * 1000;
  const bounds = geohashQueryBounds(center, radiusMeters);

  const snapshots = await Promise.all(
    bounds.map(([start, end]) =>
      firestore()
        .collection('pharmacies')
        .where('verificationStatus', '==', 'APPROVED')
        .where('status', '==', 'ACTIVE')
        .orderBy('geohash')
        .startAt(start)
        .endAt(end)
        .get(),
    ),
  );

  const unique = new Map<string, PharmacyStore>();

  snapshots.forEach(snapshot => {
    snapshot.docs.forEach(document => {
      const pharmacy = {
        id: document.id,
        ...document.data(),
      } as PharmacyStore;

      const distanceKm = distanceBetween(
        [pharmacy.location.latitude, pharmacy.location.longitude],
        center,
      );

      if (distanceKm <= radiusKm) {
        unique.set(pharmacy.id, {
          ...pharmacy,
          distanceKm,
        } as PharmacyStore);
      }
    });
  });

  return [...unique.values()].sort(
    (first, second) =>
      (first.distanceKm ?? 0) - (second.distanceKm ?? 0),
  );
}
```

Cloud Functions must calculate and save the pharmacy geohash whenever its location changes.

## 13. Medicine Availability by Pharmacy

Inventory is an optional discovery hint:

```text
pharmacies/{pharmacyId}/inventory/{medicineId}
```

```ts
export interface PharmacyInventoryItem {
  medicineId: string;
  price: number;
  status: 'AVAILABLE' | 'LOW_STOCK' | 'OUT_OF_STOCK';
  lastCheckedAt: FirebaseFirestoreTypes.Timestamp;
}
```

Rules:

1. Inventory can rank pharmacies that are more likely to fulfil an order.
2. The app must label inventory information as “Last reported availability.”
3. The app must not promise that medicine is in stock.
4. The pharmacy must confirm every requested medicine before accepting the order.
5. Only the backend transaction can assign the winning pharmacy.

## 15. Security Rules

```text
match /medicines/{medicineId} {
  allow read: if resource.data.status == 'ACTIVE';
  allow write: if isAdmin();
}

match /pharmacies/{pharmacyId} {
  allow read: if resource.data.verificationStatus == 'APPROVED'
    && resource.data.status == 'ACTIVE';
  allow write: if isAdmin() || ownsPharmacy(pharmacyId);

  match /inventory/{medicineId} {
    allow read: if get(
      /databases/$(database)/documents/pharmacies/$(pharmacyId)
    ).data.verificationStatus == 'APPROVED';
    allow write: if isAdmin() || ownsPharmacy(pharmacyId);
  }
}
```

Production rules must retain admin and pharmacy-owner checks for writes.

## 16. Loading and Error States

### Initial Loading

- Show medicine or pharmacy card skeletons.
- Do not display a full-screen spinner over an existing list.

### Pagination Loading

- Keep existing items visible.
- Show a footer loader.

### Empty Medicine List

```text
No medicines found.
Try another medicine, generic name, or brand.
```

### Empty Pharmacy List

```text
No verified pharmacies found nearby.
Increase the search radius or select another location.
```

### Error

- Explain that data could not be loaded.
- Preserve the previous successful list.
- Display a retry button.
- Log the Firebase error without personal or medical data.

## 17. Refresh Strategy

- Medicine catalogue stale time: 5 minutes
- Pharmacy list stale time: 1 minute
- Nearby pharmacy query refreshes after meaningful location movement
- Active order uses a Firestore real-time listener
- Inventory does not require a continuous listener on list screens

## 18. Completion Checklist

- Medicine list uses Firestore pagination.
- Pharmacy list only shows approved and active stores.
- Nearby pharmacies use geohash bounds and exact-distance filtering.
- Search is debounced.
- Empty, loading, pagination, error, and retry states are implemented.
- Cards are memoized.
- Callbacks passed to cards are stable.
- Firestore listeners are detached during cleanup.
- Indexes are deployed.
- Security Rules prevent unauthorized writes.
- Inventory is never presented as guaranteed stock.
- Pharmacy confirmation remains mandatory before order acceptance.
